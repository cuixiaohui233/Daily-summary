1.git分为三大区， 工作区 暂存区 保护工作区的代码不会丢失 避免版本库的代码版本混乱 版本库 2.上传代码到版本库 (1).工作区的代码上传到版本库必须要经过暂存区 这个时候查看状态，会看到文件是红色的 工作区上传到暂存区： git add <要上传的项目名> 回车 git add . ->快捷方式，把所有的代码都上传到暂存区 这个时候查看状态，会看到文件是绿色的 暂存区上传到版本库 git commit -m "添加注释" 回车 这个时候查看状态，会看到working tree clean，证明已经上传到版本库，现在工作区已经是干净的状态，这时的版本已经是最后一次提交的代码 这个时候版本库里代码、文件和暂存区的一样 (2).快捷方式，从工作区直接到版本库： git commit -a -m "注释" 回车 3.一些小命令： git status -> 查看当前状态 git log -> 查看版本信息 进入查看后 按'q'返回 4.上传到GitHub网站上： git push origin master 必须要联网 5.如果误删了某个文件： git checkout 文件夹的名字/被删除的那个文件 如果在删除的那个文件所在的文件夹里右击，出现的控制台，这个时候，就写git checkout 被删除的那个文件 就可以了 如果删除的是文件夹，这个时候就直接写git checkout 被删除的文件夹的名字 6.区别： 查看工作区和暂存区的区别： git diff 回车 查看暂存区与版本库的区别： git diff -- cached 回车 查看工作区与版本库的区别 git diff master 7.撤销： 把暂存区的代码撤销回工作区： git reset HEAD 文件名 如果工作区的代码已经放到了暂存区，并且又修改了工作区的代码 运行了撤销操作，那么这个时候**工作区修改的代码任然会保留**。 如果**撤销想还原之前在暂存区或者版本区的代码**，那么可以使用 git checkout -- 文件名来还原。 如果暂存区有代码，优先还原暂存区的代码，如果没有，还原成版本区的代码。 ***如果3区都有不同，使用了一次撤销，那么再次撤销无效。 当两个都要上传到版本库里并作为一个版本，但是忘了提交一个，这时要先将这个文件通过git add .上传到暂停区，然后通过git commit -m "注释" --amend，将之前版本库中修改的文件拽回来和这个文件的合并后在上传到版本库里，注意这个时候之前上传的版本信息已经删除了，没有了。 8删除： (1).如果手动在工作区删除后，想在版本库里也删除，就用git rm 文件名 回车 这时查看状态，会是绿色的delete，这时证明，工作区和版本库都删除了。 注意：如果要删除的文件已经在版本库，而且没有修改这个要删除的文件，那么这时没有手动在工作区删除，使用git rm 文件名也会把这个文件彻底删除，工作区和版本库都删除。如果修改了这个文件，是不允许使用git rm 文件名的 (2).如果想直接在工作区删除又在版本库里删除，就用git rm -f 文件名，这样就会工作区和版本库都删除。 (3).如果想删除整个文件夹，用git rm -rf 文件夹名字 9.恢复(回滚)： 回滚指定的文件： git checkout 版本记录 回滚的文件名 回滚到某个历史项或者指定版本 git reset -- hard 版本记录 注意：如果回滚的次数很多，有些时候会发现git log中没有想恢复的版本信息，那这个时候就可以使用git reflog 来查找操作信息。 10.多人协作： 设置多人开发的权限： 1.进入项目 2.点击settings 3.在最左边第二个列表，collaboration，可以添加协同开发人员 4.输入用户名即可 被邀请的人要确认才能合作开发这个项目。 多人协作或者自己修改的时候，有时候，你在github上修改了，又在本地修改了，这时会发生冲突，解决方案： (1).需要把远程仓库的代码拉取到本地，然后进行对比。 git fetch (2).查看冲突 git diff master origin/master (3).合并冲突 git merge origin/master (4).手动修改冲突 (5).再提交一次 (6).push 11.当没有权限还想参与别人的项目时 1.先找到想参与的项目，fork 2.把fork的项目克隆到本地 3.去修改项目文件 4.提交 5.点击 Pull requests 6.点击new Pull request 7.点击create Pull request 8.在留言板下点击create Pull request 9.等待别人的回复 12.当你收到了修改通知的时候 1.进入项目 2.点击files changed 查看修改的记录 3.如果同意点击 merge Pull request 4.点击Confirm merge 13.如何更新你叉子的项目： 如果你叉子的项目在本地没有克隆，那么，就克隆一下； 如果你叉子的项目已经克隆完了，然后这个项目有了更新，这个时候，要在GitHub官网上先进入你的项目，把这个叉子的项目删除，然后重新叉子，之后回到控制台，在你的仓库里直接写git pull 就可以完成更新。
