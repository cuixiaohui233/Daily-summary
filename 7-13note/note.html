1.git分为三大区， 工作区 暂存区 保护工作区的代码不会丢失 避免版本库的代码版本混乱 版本库 2.上传代码到版本库 (1).工作区的代码上传到版本库必须要经过暂存区 这个时候查看状态，会看到文件是红色的 工作区上传到暂存区： git add <要上传的项目名> 回车 git add . ->快捷方式，把所有的代码都上传到暂存区 这个时候查看状态，会看到文件是绿色的 暂存区上传到版本库 git commit -m "添加注释" 回车 这个时候查看状态，会看到working tree clean，证明已经上传到版本库，现在工作区已经是干净的状态，这时的版本已经是最后一次提交的代码 (2).快捷方式，从工作区直接到版本库： git commit -a -m "注释" 回车 3.一些小命令： git status -> 查看当前状态 git log -> 查看版本信息 进入查看后 按'q'返回 4.上传到GitHub网站上： git push origin master 5.如果误删了某个文件： git checkout 文件夹的名字/被删除的那个文件 如果在删除的那个文件所在的文件夹里右击，出现的控制台，这个时候，就写git checkout 被删除的那个文件 就可以了 如果删除的是文件夹，这个时候就直接写git checkout 被删除的文件夹的名字 6.区别： 查看工作区和暂存区的区别： git diff 回车 查看暂存区与版本库的区别： git diff -- cached 回车 查看工作区与版本库的区别 git diff master 7.撤销： 从暂存撤销到工作区: git reset 文件路径 从版本库撤销到工作区： git checkout -- 文件路径 当修改文件和新建文件都要上传到版本库里并作为一个版本，这时上传到版本库时会上传修改的，这个新建的上传不上去或者上传后和修改的不在一个版本库里，这时要先将这个新建的通过git add .上传到暂停区，然后通过git commit -m "注释" --amend，将之前版本库中修改的文件拽回来和新建的合并后在上传到版本库里，注意这个时候之前修改的所在的版本信息已经删除了，没有了。
